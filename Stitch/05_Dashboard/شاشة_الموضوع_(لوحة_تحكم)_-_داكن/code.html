Advanced State Management in 2024: Moving Beyond Redux?

calendar_today Oct 12, 2023 schedule 2:45 PM check_circle Solved
Original Post

Sarah Jenkins

Senior Developer

I've been working on a large scale enterprise application for the last 6 months, and we're hitting some significant bottlenecks with our current global state implementation.

We started with Redux Toolkit, which was great for boilerplate reduction, but the amount of re-renders we are seeing in the profiler is concerning, especially with large datasets.

const heavySelector = createSelector(
  (state) => state.largeData,
  (data) => data.filter(complexFilter)
);

I am considering migrating parts of the state to Zustand or even Jotai for more atomic updates. Has anyone handled a migration like this on a live production app? Specifically interested in:

  • Performance gains vs refactoring effort
  • DevTools experience comparison
  • Team onboarding curve
3 Replies
Alex Chen verified
2 hours ago

We switched to Zustand last year. The biggest win was definitely the atomic updates without needing memo everywhere. The boilerplate reduction is insane compared to Redux.

Just make sure you properly separate your stores by domain logic, otherwise you end up with one massive store file that's hard to maintain.

check_circle Solution
Mike Ross Staff

Actually, before you migrate everything, have you tried using the new Context selectors or simply optimizing your Redux slices? Sometimes the issue isn't the library but the data structure normalization.

That said, Jotai is excellent if you have a lot of derived state that depends on other atoms. It feels very "React-native".
