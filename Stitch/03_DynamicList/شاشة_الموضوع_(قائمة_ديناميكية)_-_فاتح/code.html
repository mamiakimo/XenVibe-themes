Hi everyone,

We are planning to migrate our large community from a legacy XenForo 1.5 installation to a completely headless modern stack using React and the new XenForo 2.x API.

I have a few concerns regarding the data integrity of custom fields and attachments. Has anyone here done a similar "Big Bang" migration?

  • How did you handle 3rd party addon data?
  • Did you use the official CLI importers?
  • What was the downtime like?

Any advice or resources would be greatly appreciated!

MikeDev Developer • 1h ago

I strongly recommend using the CLI importers. They are much faster and handle memory management way better than the web-based ones, especially for large attachments.

For 3rd party add-ons, you might need to write custom mapping scripts if the schema has changed drastically. Check out the XenForo resource manager for some community migration scripts.

AlexT • 45m ago

MikeDev said:

This exactly. We had a gallery addon that was unsupported in 2.x. We ended up writing a Laravel script to pull the images and re-insert them into the official XF Media Gallery. It took a weekend but was worth it.
