Hi everyone, I'm trying to fine-tune our XenForo instance running on Nginx + PHP-FPM. We've been seeing some high load averages during peak times.

Here is my current nginx.conf snippet for the PHP location block. Does anyone spot any obvious issues or missing optimizations?

location ~ \.php$ {
  fastcgi_pass unix:/var/run/php/php8.1-fpm.sock;
  fastcgi_index index.php;
  include fastcgi_params;
  fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
  # Is this buffer size enough?
  fastcgi_buffers 16 16k;
  fastcgi_buffer_size 32k;
}

Any help would be appreciated!

You definitely want to increase those buffers if you are serving large pages. Also, make sure you have opcache enabled in your php.ini. That usually gives the biggest performance boost for XenForo.

The buffers look okay for standard requests, but I'd suggest adding:

fastcgi_busy_buffers_size 64k;

Also, check your pm.max_children settings. If that's hitting the limit, Nginx will throw 502 errors regardless of your buffer settings.
